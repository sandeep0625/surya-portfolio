Embedded Power + IoT

Dynamic Load Balancer for Satellite Microgrid System

Designed an embedded power management system for a satellite microgrid using a SAMW25 MCU, custom PCBA, and FreeRTOS firmware. The system monitored power channels, tracked battery state, controlled loads through MOSFETs, and published telemetry to a dashboard.

Click to expand technical breakdown
Watch Demo Video

System goal

The project acted like a small power-management controller for a satellite microgrid. It monitored electrical channels, tracked battery state, made load-control decisions, and pushed telemetry to a dashboard so the system state could be observed remotely.

Hardware layer

The design used a SAMW25-class embedded platform, current/voltage sensing, battery telemetry, MOSFET-based load switching, and regulated rails. The hardware challenge was making the power path measurable and controllable instead of treating it as a black box.

Firmware layer

The firmware was organized around FreeRTOS-style tasks for polling sensors, managing load states, handling CLI/debug commands, publishing telemetry, and supporting update paths. I2C devices and system health checks were central to the control loop.

System integration

MQTT and Node-RED turned the embedded board into a visible system. Instead of only proving the firmware locally, the dashboard made voltage, current, battery, and load state easier to inspect during demos and debug.

Robotics + Bare-Metal Firmware

MimicArm: Gesture-Controlled Robotic Arm

Built a 4-DOF gesture-controlled robotic arm using an ATmega328PB, MPU6050 IMU, flex sensors, and servo motors. The system converted hand motion and flex input into real-time servo actuation.

Click to expand technical breakdown
Watch Demo Video

System goal

The goal was to translate hand and wrist motion into robotic-arm movement. The project combined sensing, gesture interpretation, timing-sensitive actuation, and a control structure that could respond quickly enough to feel natural.

Sensing

The MPU6050 provided motion/orientation data over I2C, while flex sensors were read through ADC channels to estimate gesture input. The system had to smooth noisy sensor values without making the arm feel delayed.

Actuation

Servo outputs were controlled through PWM signals. The firmware mapped gesture states into servo positions while keeping updates predictable and avoiding sudden motion jumps.

Firmware structure

The bare-metal C code separated acquisition, processing, control mapping, and debug output. Serial logs helped tune thresholds and verify that sensor values matched the arm response.
